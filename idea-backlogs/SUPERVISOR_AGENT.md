# 슈퍼바이저 에이전트 (AEP Supervisor Agent)

## 1. 개요
단순한 채팅 창이 아니라, AEP 아키텍트처럼 전체 데이터 흐름을 시각화하고 최적의 **'실행 마인드맵'**을 제시하는 중앙 컨트롤러입니다. 도메인별로 세분화된 전문 에이전트들을 지휘하여 복잡한 API 체인을 완수합니다.

## 2. 핵심 아키텍처 (Reasoning Loop)
에이전트는 다음의 4단계 루프를 거쳐 작동합니다:

1.  **메뉴 제시 (Contextual Menu)**: 실행 환경을 분석하여 최적의 경로를 메뉴로 제안합니다.
2.  **마인드맵 수립 (Visual Planning)**: 사용자가 선택한 목표에 대해 리소스 간 관계가 포함된 시각적 마인드맵(DAG)을 생성합니다.
3.  **사용자 승인 (Human-in-the-loop)**: 수립된 계획과 예상되는 변화를 사용자에게 리치 텍스트(Rich Text)로 보여주고 실행 승인을 받습니다.
4.  **실행 및 모니터링 (Execution & Healing)**: 커맨드를 하나씩 실행하며, 실패 시 이유를 분석하여 수정된 계획을 다시 제안합니다.

## 3. 에이전트 레이어 세분화 (Agent Roles)
효율적인 작업 수행을 위해 슈퍼바이저는 다음의 전문 에이전트에게 작업을 위임합니다:

- **Schema Architect**: XDM 모델링 및 표준 필드 그룹 매핑 전문
- **Identity Expert**: 식별자(Identity) 전략 및 네임스페이스 구성 전문
- **Data Engineer**: 데이터셋 생성, 배치 수집 및 데이터플로우 관리 전문
- **Governance Officer**: 데이터 권한 및 규정 준수 체크 전문

## 3. 상세 설계 요소

### A. 의도 파싱 및 컨텍스트 주입
- **Context Awareness**: 현재 샌드박스의 상태, 로컬 파일 목록, 이전 커맨드 이력을 LLM 프롬프트에 포함하여 정확도를 높입니다.
- **Tool Definition**: CLI가 제공하는 모든 명령어를 LLM이 이해할 수 있는 '도구(Tools)' 형태로 정의합니다.
- **Dynamic Menu**: 입력된 파일 확장자나 데이터 내용에 따라 메뉴 구성을 동적으로 변경합니다 (예: CSV 파일 입력 시 '스키마 생성 플랜' 우선 노출).

### B. 메뉴 구성 예시 (Plan Selection)
사용자가 `aep assistant --file sales.csv` 실행 시:
1. **[추천] 신규 마이그레이션 플랜**: 스키마 분석부터 업로드까지 전체 자동화
2. **데이터셋 검증 플랜**: 기존 스키마와의 정합성만 체크
3. **XDM 템플릿 변환**: 샘플 데이터를 바탕으로 JSON 템플릿만 생성
4. **자유 대화**: 특정 질문 직접 입력

### C. 단계별 실행 계획 예시
- **사용자 입력**: "이 `sales.csv` 파일로 판매 내역 스키마 만들고 AEP에 올려줘."
- **에이전트 계획**:
    1.  `aep migrate analyze sales.csv`: 파일 구조 분석
    2.  `aep schema create --template ...`: 분석된 결과를 바탕으로 스키마 생성
    3.  `aep dataset create --schema-id ...`: 생성된 스키마로 데이터셋 생성
    4.  `aep ingest --file sales.csv --dataset-id ...`: 데이터 최종 업로드

### C. 안전 장치 (Safety Rails)
- **Dry-run 우선**: 모든 계획은 실제 실행 전 `dry-run` 결과를 시뮬레이션하여 사용자에게 보고합니다.
- **롤백 제안**: 중간 단계에서 실패할 경우(예: 데이터셋 생성 실패), 이미 생성된 스키마를 삭제할지 혹은 유지할지 사용자에게 묻습니다.

## 4. 사용자 인터페이스 (UX) 디자인

### 4.1 인터랙티브 메뉴 선택 시나리오 (TUI)

사용자가 `aep assistant`를 실행하면 에이전트가 로컬 파일과 샌드박스 상태를 분석한 후, 방향키로 선택 가능한 메뉴를 제시합니다.

```bash
> aep assistant

🔍 로컬 환경 분석 중... 
✓ 'customers.csv' 파일 발견 (2.4 MB)
✓ AEP 샌드박스 'prod' 연결됨

? 수행할 작업을 선택하세요: (방향키로 이동, Enter로 선택)
❯ [추천] 신규 마이그레이션 플랜 (customers.csv 분석 후 스키마/데이터셋 생성)
  [진단] 기존 데이터셋 데이터 검증 (기존 스키마와 customers.csv 비교)
  [템플릿] XDM 샘플 JSON 생성
  [관리] 현재 샌드박스 리소스 정리 (미사용 스키마 등 삭제)
  [채팅] 에이전트와 자유 대화
```

**상세 동작:**
1.  **커서 이동**: 위/아래 방향키를 누르면 `❯` 표시와 배경색이 이동합니다.
2.  **동적 설명**: 각 메뉴 선택 시 하단에 해당 플랜의 상세 설명이 표시됩니다.
    - *예: "신규 마이그레이션 플랜 선택 시: AI가 CSV 구조를 분석하여 XDM 스키마를 설계하고 데이터셋 생성까지 자동 가이드합니다."*
3.  **선택 완료**: Enter를 누르면 다음 단계인 '실행 계획 시각화' 단계로 진입합니다.
