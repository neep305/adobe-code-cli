# AEP 마인드맵 (AEP Mindmap)

## 1. 개요
AEP의 샌드박스 내에 존재하는 수많은 리소스(XDM 스키마, 데이터셋, 아이덴티티 네임스페이스, 데이터플로우) 간의 연결 고리를 시각화하여 데이터 에코시스템의 전체 지도를 제공합니다.

## 2. 주요 시각화 요소
1. **스키마-데이터셋 매핑**: 특정 스키마가 어떤 데이터셋들에서 사용되고 있는지 연결.
2. **아이덴티티 리니지 (Identity Lineage)**: 여러 데이터셋에 걸쳐 공유되는 Identity 필드를 중심으로 데이터가 어떻게 연결되는지 시각화.
3. **데이터 소스-타겟 흐름**: Source Connection에서 시작하여 Dataset을 거쳐 Destination으로 나가는 데이터의 흐름 표시.
4. **필드 레벨 의존성**: 복잡한 XDM 구조 내에서 필수 필드와 식별자 필드를 강조 표시.
5. **진행 가이드 마인드맵**: 작업 전/후의 리소스 변화를 시뮬레이션하여 다음 단계를 시각화.

## 3. 핵심 기능
- `aep mindmap generate`: 현재 샌드박스의 리소스를 분석하여 마인드맵 데이터 생성.
- `aep mindmap next`: 현재 작업 상태를 기반으로 다음에 필요한 리소스 연결을 마인드맵으로 제안.
- `aep mindmap view --format html`: 브라우저를 통해 인터랙티브한 노드 그래프 시연.
- `aep mindmap export --format mermaid`: 기술 문서용 정적 이미지 내보내기.
- **AI 인사이트**: "이 데이터셋은 스키마와 연결되어 있지만 최근 30일간 유입된 데이터가 없습니다"와 같은 고립된 노드 탐지.

## 4. 기대 효과
- **임팩트 분석**: 스키마 변경 시 영향을 받는 모든 데이터셋과 데이터플로우를 즉시 파악.
- **데이터 거버넌스**: 데이터가 어떤 경로로 흐르고 어디서 식별되는지 한눈에 검토.
- **온보딩 가속화**: 신규 프로젝트 참여자가 기존 AEP 설정을 빠르게 이해.
